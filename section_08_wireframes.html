S01 — Search
Filter chips surface specialty, location, and availability — no multi-step form needed.
S02 — Profile
Key stats (rating, years, reviews) above the fold so trust is earned before the CTA.
S03 — Booking
Date strip + slot grid collapses 3 screens into one — 2 taps to confirm a time.
S04 — Payment
Summary card stays anchored throughout payment so users never lose context on what they're booking.
S05 — Chat
Async messaging keeps the care loop alive after the visit without forcing a new appointment.
Single-scroll layouts
Every screen is designed to work without horizontal navigation. All critical content fits in one continuous scroll.
Progressive disclosure
Filters, date pickers, and payment fields appear contextually — reducing cognitive load at each decision point.
Persistent context
Booking summary stays visible through payment so the user always knows exactly what they're confirming.
✕ Rejected direction
Split flow: search → separate booking screen
Early wireframes had doctor profile and slot selection on two separate screens. The structure felt logical but added a back-and-forth that broke the decision momentum. Merged into one continuous scroll — profile above, slots below, CTA anchored at the bottom.
